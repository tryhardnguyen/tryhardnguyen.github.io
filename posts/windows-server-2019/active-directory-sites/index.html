What is an Active Directory Sites?

Basically it’s an physical location. For example, McDonalds is a global company. Chances are they have servers in America, Korea, Japan, Canada. Each of these location are sites.

Reason to create an Active Directory Sites:

  1. Replication
  2. Service Location
Replication allows data to be backup so that in cases where one server crash, one server can take over.

Service Location allows one site to focus on one area, and cases where one site crashes, the other sites can take over.

Create an Active Directory Sites

In server manager, go to “Active Directory Sites and Services”.

Now, right-click on “Sites” and click on “New Site”. Then a new menu should pop up. We will name our new site. For this example, I will name my site Canada because the company located in Canada. Then click OK.

Now, we will click on “Subnets” and “New Subnet”. Then we will assign an IP range for the site that located in Canada. For this example, I will use 192.168.206/24. Basically anything that falls in this range will be treated part of Canada site.
